# CLI-Net

CLI-Net — это MVP терминальной социальной сети, работающей исключительно через SSH. В проекте нет ни веб-интерфейса, ни мобильного приложения. Пользователи подключаются к серверу через привычный SSH-клиент, проходят аутентификацию по своим публичным SSH-ключам и общаются через функциональный TUI (Text User Interface).

Проект написан на Go и построен на базе инструментов от [Charmbracelet](https://charm.sh/): сервера **Wish**, интерактивного фреймворка **Bubble Tea** и библиотеки стилей **Lip Gloss**. Для работы с данными используется встраиваемая база данных **SQLite**.

---

## 🌟 Основные механики

### 1. Беспарольная аутентификация
При подключении сервер автоматически перехватывает публичный SSH-ключ пользователя. Если пользователь подключается впервые, для него создается новый анонимный профиль в базе данных (например, `anon_4F2A`), и он считается неверифицированным.

### 2. Квест для новичков (Onboarding)
Новые неверифицированные пользователи попадают в `QuestView` (специальный экран инициализации). Им предлагается декодировать простую строку из Base64. В нашем случае это секрет — `let me in`. После успешного ввода пользователь получает статус верифицированного и переходит к основному чату.

### 3. Real-Time Feed (Общение в реальном времени)
Под капотом работает глобальный PubSub-брокер, который использует мьютексы и стандартные горутины/каналы Go. Как только вы отправляете сообщение в TUI, оно сохраняется в SQLite и мгновенно рассылается всем активным SSH-сессиям, которые находятся в этом же канале.

### 4. Поддержка копирования сниппетов (OSC 52)
Поддерживается стандарт escape-последовательностей OSC 52. В главном меню вы можете нажать `Ctrl+Y`, чтобы скопировать последнее сообщение напрямую в локальный буфер обмена вашего компьютера (минуя SSH сессию).

## 🏗 Архитектура проекта

- **`/cmd/clinet/` (main.go)** — Точка входа в приложение.
- **`/internal/ssh/`** — Настройка SSH-сервера (Wish), перехват публичных ключей, аутентификация.
- **`/internal/tui/`** — Все модели и логика TUI (Bubble Tea). Содержит экраны (Квест и Чаты) и обновление состояния.
- **`/internal/db/`** — Схемы SQLite, миграции данных, CRUD-операции.
- **`/internal/pubsub/`** — Гоночно-безопасный (concurrent-safe) брокер сообщений для широковещательной рассылки.

## 🚀 Как запустить

### Шаг 1. Запуск сервера
Убедитесь, что у вас установлен Go версий 1.21+.

Перейдите в директорию проекта и загрузите зависимости:
```bash
go mod tidy
```

Запустите сервер (база данных `clinet.db` будет создана в корне автоматически при первом старте):
```bash
go run ./cmd/clinet/main.go
```
*Сервер по умолчанию запустится на порте `2222`.*

---

### Шаг 2. Подключение клиента
Откройте **новое окно** или вкладку вашего эмулятора терминала и подключитесь локально:
```bash
ssh -p 2222 localhost
```

### Шаг 3. Использование TUI
1. При первом входе вы увидите квест: `Welcome to CLI-Net! Prove you are worthy.` Введите ответ `let me in` и нажмите Enter.
2. В основном меню навигация интуитивно понятна:
   - Выбор каналов (переключение между `#general`, `#linux` и т.д.) осуществляется клавишей **`Tab`**.
   - Введите текст в нижней части и нажмите **`Enter`**, чтобы отправить сообщение.
   - Скопировать последнее сообщение канала в ваш системный буфер обмена можно клавишей **`Ctrl+Y`**.
   - Для выхода нажмите **`Ctrl+C`** или **`Esc`**.
