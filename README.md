# CLI-Net v1.0

CLI-Net — это современная терминальная социальная сеть, работающая исключительно через SSH. В проекте нет веб-интерфейса, мобильного приложения или регистрации по паролю. Вы подключаетесь к серверу через привычный терминал, проходите аутентификацию по своим публичным SSH-ключам и общаетесь через функциональный, полноэкранный 4-х панельный TUI (Text User Interface).

Проект написан на Go и построен на базе передовых терминальных библиотек от [Charmbracelet](https://charm.sh/): 
- **Wish** (SSH Сервер)
- **Bubble Tea** (Интерактивная TUI архитектура)
- **Lip Gloss** (Стилизация UI компонентов)
- **Glamour** (Рендеринг Markdown в терминале)

Для хранения данных используется встраиваемая база **SQLite** с включенным журнальным модом (WAL) для конкурентного чтения и записи.

## 🌟 Основные фичи v1.0

### 1. 4-Панельный Интерфейс (UI)
Экран динамически подстраивается под размеры вашего терминала:
- **`CHANNELS` (Левая Панель):** Навигация по комнатам (`#general`, `#linux`, `#bash-magic`). Между каналами переключение осуществляется клавишей `Tab`.
- **`ONLINE` (Правая Панель):** Живой список пользователей, находящихся с вами в одной комнате прямо сейчас (отсортированный по алфавиту для удобства).
- **`FEED` (Центральная Панель):** Чат со встроенным **рендрингом Markdown**. Пишите куски кода или ставьте **жирный** текст — сервер отрендерит это прямо в окне.
- **`INPUT` (Нижняя Панель):** Интерактивная область для ввода команд и текста сообщений.

### 2. Механика Присутствия и Системные Уведомления
Базовый Pub/Sub брокер полностью контролирует сессии пользователей. Если кто-то подключается к вашему каналу, вы увидите системное сообщение: `Server: anon_1a2b joined the channel`. При отключении или смене канала другим пользователем — он мгновенно пропадет из бокового списка.

### 3. Авторизация и Учетные Записи
Вся аутентификация происходит автоматически по вашему приватному SSH ключу `ED25519` / `RSA`. При первом входе система генерирует для вас случайный ник и закрепляет за вами **уникальный HEX Цвет** (например, золотой или фиолетовый), который используется во всех чатах.

### 4. Продвинутые Slash-команды (`/`)
Команды можно вводить прямо в чат:
- `/nick <ИМЯ>` — Сменить свой глобальный ник. Он моментально обновится в базе и панели онлайна.
- `/clear` — Локально очистить вывод чата в вашем экране.
- `/help` — Вывести в чат справочную информацию по горячим клавишам.

### 5. Иерархия Ролей и Модерация (Op Mode)
В системе заложена прогрессивная модерация:
- **Владелец (`owner`) 👑:** Эта роль выдается автоматически первому зарегистрированному юзеру на сервере. Владельца нельзя забанить или кикнуть. Он имеет право назначать админов и передать права владельца:
  - `/op <НИК>` — Назначить пользователя администратором.
  - `/deop <НИК>` — Снять права администратора.
  - `/setowner <НИК>` — Передать статус Владельца другому, автоматически понижая себя до админа.
- **Администратор (`admin`) ★:**
  - `/kick <НИК>` — Принудительно отключает пользователя от SSH сервера (обрыв сессии).
  - `/ban <НИК>` — Кикает юзера и прописывает перманентный бан (отклоняет попытки реконнекта).
  - `/unban <НИК>` — Снимает бан с юзера по никнейму.
- **Пользователь (`user`):**
  - `/op me` (только если вы владелец) или другие пользовательские бэйдж-команды.

### 6. Встроенное копирование OSC 52
Нажав `Ctrl+Y`, вшитая escape-последовательность мгновенно скопирует текст последнего сообщения из нутров удаленного сервера напрямую в ваш ОС буфер обмена (Windows, Mac, Linux). Возможны ограничения со стороны некоторых SSH клиентов.

## 🚀 Как запустить

### Шаг 1. Сборка Сервера
Убедитесь, что у вас установлен `Go 1.21+`.
Загрузите зависимости:
```bash
go mod tidy
```

Соберите бинарный файл:
```bash
go build -o clinet ./cmd/clinet
```

Сервер автоматически установлен как фоновый `systemd` сервис на вашей машине.
Для его управления используйте стандартные команды Linux:
```bash
sudo systemctl status clinet
sudo systemctl restart clinet
```
*Сервер работает на порту `2222`, логи записываются в `server.log` в корне проекта.*

---

### Шаг 2. Подключение
В корне проекта лежит удобный bash-скрипт клиента. Просто выполните его:
```bash
./clinet-connect
```
(или подключитесь напрямую через `ssh -p 2222 localhost`).

Как только вы нажмете `Ctrl+C` или `Esc` находясь в TUI — ваш SSH-клиент закроется, но сервер в фоне продолжит бесперебойную работу!
